Concept

What does “Mobile‑First” actually mean?

  • Prioritize the smallest screen & simplest journey first.
  • Ship the core value fast; enhance progressively for wider screens.
  • Design for thumb zones, short attention, and spotty networks.

Business lens: faster MVP, clearer message, better conversions.

Responsive Mental Model

  1. Flow (stack vertically)
  2. Wrap (allow items to wrap)
  3. Re‑order (change priority visually)
  4. Enhance (add columns, larger type, extras)

Strategy

Why it matters for startups

Benefits → Impact

  • Time‑to‑market → mobile web MVP in days, not months.
  • Conversion → frictionless checkout/forms on phones.
  • Cost → one responsive build across devices.
  • SEO → Google prioritizes mobile experience.
  • Accessibility → reach more users globally.

Quick Audit (class prompt)

  1. Above‑the‑fold: value prop + CTA visible?
  2. Tap targets ≥ 44px?
  3. Page weight under ~1–2MB?
  4. Works on 3G/slow network?

If any fail on mobile, you’re leaking revenue.

When to choose what?

  • Landing fast: Next.js + Tailwind or Webflow.
  • App both stores: React Native or Flutter.
  • Offline heavy: PWA with caching.
  • Content team: Headless CMS + responsive frontend.
Concept

Progressive Web Apps (PWA) — web that feels like an app

  • Installable on phone (no store required)
  • Offline caching for repeat visits
  • Push notifications (with permission)

Case

Case: Instagram traffic → mobile checkout

Scenario

A fashion startup’s traffic is 90% mobile via Instagram. Desktop site works, but mobile bounce is high.

  • Problem: checkout friction on phones.
  • Fix: compress images, one‑page checkout, Apple/Google Pay, sticky CTA.

Estimated uplift: 0%

Rule of thumb

  • Above‑the‑fold = promise + proof + CTA.
  • Minimize typing; prefer selection & native wallets.
  • Load critical content first; defer the rest.
