HTML5 Semantic Elements Quiz

Test your understanding of semantic HTML! Answer these questions to reinforce what you've learned.

Question 1: What is Semantic HTML?

  • A) HTML that uses colors and styling
  • B) HTML elements whose names describe their meaning/role in the document
  • C) HTML written by AI tools
  • D) HTML that only works in Chrome

Answer: B ‚Äì Semantic elements like <header>, <nav>, <article> convey meaning to browsers and assistive technologies.

Question 2: Which is NOT a semantic element?

  • A) <header>
  • B) <div>
  • C) <article>
  • D) <footer>

Answer: B ‚Äì <div> is a non-semantic, generic container element.

Question 3: When should you use <article> vs <section>?

  • A) Use <article> for self-contained, reusable content (blog post, card)
  • B) Use <section> for thematic groupings with a heading
  • C) Both A and B
  • D) They are exactly the same

Answer: C ‚Äì <article> is for standalone content, <section> groups related content thematically.

Question 4: Why is the <main> element limited to one per page?

  • A) It's a browser limitation
  • B) It represents the unique main content of the document
  • C) Multiple <main> tags will break the page
  • D) It makes styling easier

Answer: B ‚Äì <main> identifies the primary content, which should be unique per page.

Question 5: Give two reasons semantic HTML improves SEO.

Sample Answer:
1. Better structure parsing: Search engines can better understand the hierarchy and importance of content.
2. Improved accessibility: Screen readers and assistive tech provide better navigation, which search engines reward.
