What is a Webpage?

A webpage is more than just a single file. It's a collection of parts that work together.

Analogy: A house is built with a frame (HTML), styled with paint & furniture (CSS), and given electricity & plumbing (JavaScript).

The Building Blocks

  • HTML (HyperText Markup Language): The structure of the page. It's the skeleton that provides the content and meaning (headings, paragraphs, links).
  • CSS (Cascading Style Sheets): The style. It controls the colors, fonts, layout, and visual appearance of the page.
  • JavaScript: The behavior. It makes the page interactive and dynamic.

The Client‚ÄìServer Model

The Client: Your web browser (Chrome, Firefox, Safari) that requests information.

The Server: A powerful computer that provides the website's files.

The Request‚ÄìResponse Cycle

  • Request: You type a URL (e.g., google.com) into your browser. The browser sends an HTTP request to the server.
  • Lookup: The browser finds the server's address.
  • Response: The server finds the requested files (HTML, CSS, JS) and sends them back to the client.
  • Render: Your browser assembles the files and renders the webpage for you to see.

Introducing HTTP

HTTP (Hypertext Transfer Protocol) is the "language" that the client and server use to communicate.

It's a set of rules for transferring hypermedia documents, like HTML.
