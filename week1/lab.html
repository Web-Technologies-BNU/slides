How the web works (in 20 seconds)

Browser (Chrome, Safari) sends a request → Server sends back files (HTML, CSS, JS, images).

For now, we’ll open our file locally by double‑clicking it.

Request:  GET /index.html
Response: 200 OK (HTML + CSS + JS)

Setup (5 minutes)

  1. Create a folder: web101
  2. Open in VS Code (or any editor)
  3. Create a file: index.html
  4. Paste the starter template (next slide)
  5. Double‑click index.html to open in your browser

Starter template

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>My First Page</title>
  <style>/* quick styles go here */ body { font-family: system-ui; } </style>
</head>
<body>
  <h1>Hello, world!</h1>
  <p>This is my first HTML page.</p>
</body>
</html>

Save and refresh the browser tab to see changes.

Page anatomy: <head> vs <body>

<head>

  • <meta> — settings (charset, viewport)
  • <title> — tab title
  • <style> or <link> — CSS
  • <script> — optional JavaScript includes

<body>

  • Visible content: headings, paragraphs, images, links, lists
  • What users actually see and interact with

<head> essentials

<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Page Title</title>
  <style>
    body { font-family: system-ui; margin: 40px; }
    h1 { color: #0ea5e9; }
  </style>
</head>

Tip: use <style> for quick demos; switch to a separate CSS file later.

<body> basics

<body>
  <h1>Welcome</h1>
  <p>This is a paragraph.</p>
  <ul>
    <li>One</li>
    <li>Two</li>
  </ul>
</body>

Images with <img>

<img src="images/logo.png" alt="Company logo" width="200">

Attributes: src (path to file), alt (text for screen readers/when image fails), optional width/height.

Keep images inside an images/ folder for neatness.

Links with <a>

<a href="https://example.com" target="_blank">Visit Example</a>

Attributes: href (destination), target (_self = same tab, _blank = new tab)

Use descriptive link text. Avoid: “click here”. Prefer: “Read our pricing”.

Adding quick styles

<style>
  body { font-family: system-ui; margin: 40px; background: #f7fafc; }
  .btn { background: #111827; color:#fff; padding: 10px 14px; border-radius: 8px; }
</style>

Later: move CSS to styles.css and link it with <link rel="stylesheet" href="styles.css">

Mini‑exercise (10 minutes)

  1. Add a heading with your name.
  2. Write a short paragraph about you.
  3. Add one image and one link to a site you like.
  4. Change heading color using <style>.

Save the file and refresh the browser tab to preview.

Cheat sheet

<!DOCTYPE html>
<html lang="en">
<head> ... </head>
<body> ... </body>
</html>
  • <head>: meta, title, styles, scripts
  • <body>: everything visible
  • <img>: images with src + alt
  • <a>: links with href

From template → resume page

We’ll build a simple resume page with headings, paragraphs, image, and links.

Folder layout

web101/
  index.html
  images/
    headshot.jpg
  styles.css (optional)

Resume scaffold

<body>
  <header>
    <h1>Your Name</h1>
    <p>Email · City · Portfolio link</p>
  </header>
  <main>
    <h2>Experience</h2>
    <ul>
      <li>Role @ Company — one‑line impact</li>
    </ul>
    <h2>Skills</h2>
    <ul><li>HTML</li> <li>CSS</li> <li>JS</li></ul>
  </main>
</body>

Image + link in resume

<img src="images/headshot.jpg" alt="Your Name headshot" width="140">
<p>Portfolio: <a href="https://yourdomain.com" target="_blank">yourdomain.com</a></p>

Quick styles for resume

<style>
  body { max-width: 860px; margin: 48px auto; padding: 0 20px; font-family: system-ui; }
  h1 { color: #22d3ee; margin-bottom: 0; }
  header p { color: #475569; }
  h2 { margin-top: 28px; }
</style>

DevTools: your superpower

Right‑click → Inspect. Edit HTML/CSS live in the Elements/Styles panels. Refresh resets to your file.

Shortcut: Cmd/Ctrl + Shift + I

Exercise (in class)

  1. Finish your resume page (name, paragraph, image, link).
  2. Add one custom style (color or spacing).
  3. Show your neighbor. Give one suggestion each.

Wrap‑up & homework
