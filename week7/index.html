JavaScript Core

== vs === ‚Äî What‚Äôs the Difference?

JavaScript can be ‚Äúhelpful‚Äù and convert types for you. That‚Äôs what == does. === is stricter and compares both value and type.

Definitions
  • == (loose equality)
    Compares values after doing type conversion.
  • === (strict equality)
    Compares value and type. No conversion.
  • Assignment is = (don‚Äôt mix it up with comparison).

In modern JavaScript / React codebases, developers almost always use === and avoid == unless they really know what they are doing.

Quick examples 
5 == "5"     // true   (string "5" converted to number)
5 === "5"    // false  (number vs string)

0 == false   // true   (false converted to 0)
0 === false  // false  (number vs boolean)

null == undefined   // true   (special case)
null === undefined  // false  (different types)
Examples

How JavaScript Actually Compares Values

Let‚Äôs see concrete examples of == and === side by side.

Equality Comparison Table
Expression Result Why?
5 == "5" true String converted to number, 5 == 5.
5 === "5" false Number vs string, types differ.
0 == false true false becomes 0, 0 == 0.
0 === false false Number vs boolean, types differ.
"" == 0 true Empty string converted to 0.
"" === 0 false Again, string vs number.
null == undefined true Special loose equality rule between these two.
null === undefined false Different types.

Rule of thumb for students: use === and !==. If you reach for ==, you should be able to explain why.

Best Practices

When To Use Which?

We keep it simple: strict equality by default, with a few special cases.

Recommended Style
  • Use === for comparisons in your logic and React conditions.
  • Use !== for ‚Äúnot equal‚Äù.
  • Avoid tricks like if (value == null) early on.
  • Be explicit: compare the exact value and type you expect.

This will reduce weird bugs when you fetch JSON and compare things like "1" vs 1.

React-friendly examples 
// Good: explicit types and strict checks
const [count, setCount] = useState(0);

if (count === 0) {
  console.log("Nothing yet.");
}

// Avoid this early on:
if (count == "0") {
  // Works but confusing: number vs string
}
React + JSON

Fetching JSON in React

In a React app, we usually: fetch JSON from an API, put it in state, and then render it.

Simple fetch example 
import React, { useEffect, useState } from "react";

function UsersList() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function loadUsers() {
      const response = await fetch("/api/users"); // Your Node API
      const data = await response.json();        // JSON ‚Üí JS object/array
      setUsers(data);
      setLoading(false);
    }

    loadUsers();
  }, []);

  if (loading) {
    return <div>Loading users...</div>;
  }

  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
Key Ideas
  • useState keeps the JSON in component memory.
  • useEffect runs after the first render to fetch data.
  • fetch() gets data from your Node / public API.
  • response.json() converts JSON string ‚Üí JavaScript object/array.
  • map() is used to loop over arrays and render JSX.
Reading JSON

Accessing JSON Fields in React

JSON is just nested objects/arrays. In React, you use dot and bracket notation like normal JavaScript.

Example JSON 
const data = {
  user: {
    name: "Sara",
    age: 21,
  },
  courses: [
    { id: 1, title: "Web Tech", grade: "A" },
    { id: 2, title: "Databases", grade: "B+" },
  ],
};
How to access in React
JSX snippets
// Inside a React component's return:
<div>
  <h2>{data.user.name}</h2>
  <p>Age: {data.user.age}</p>

  <h3>Courses</h3>
  <ul>
    {data.courses.map((course) => (
      <li key={course.id}>
        {course.title} ‚Äî Grade: {course.grade}
      </li>
    ))}
  </ul>
</div>
Watch Out For
  • Undefined data: API might not have loaded yet, so always handle loading states.
  • Type issues: API may send numbers as strings. Use === correctly when comparing.
  • Deep nesting: Sometimes you need to check if something exists: data && data.user && data.user.name (or optional chaining later).

For Week 7, keep JSON structures simple and shallow.

React Basics

What Is React, Really?

React is a JavaScript library for building user interfaces. It helps you build apps as small pieces called components.

Core Ideas
  • Component-based: UI is broken into small, reusable pieces.
  • Declarative: You describe ‚Äúwhat UI should look like for this state‚Äù.
  • State: Data that can change over time in your component.
  • Props: Inputs to components (like function parameters).

Think of each component as a ‚Äúmini function‚Äù that returns HTML (JSX) based on its data.

A very tiny React component 
function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Use it like this:
<Welcome name="BNU Students" />

JSX lets you write HTML-like syntax inside JavaScript. It is not a string, it compiles to React.createElement calls.

React State

Simple React Component With State

State lets a React component ‚Äúremember‚Äù values between renders.

Counter example 
import React, { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0); // initial value: 0

  function increment() {
    setCount(count + 1);
  }

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={increment}>Click me</button>
    </div>
  );
}
Important Concepts
  • useState(0) gives you a pair: [count, setCount].
  • count is the current value; setCount updates it.
  • When you call setCount, React re-renders the component.
  • Using === in conditions: {count === 0 && "Start clicking!"}

You‚Äôll use the same pattern when you store JSON from APIs in state and display it.
